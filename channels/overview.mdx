# Channels Overview

Channels represent where you sell your products. Tether allows you to manage multiple sales channels, track performance by channel, and configure channel-specific settings.

## What is a Channel?

A channel is a sales avenue for your products:

- **Direct-to-Consumer**: Shopify, your website
- **Marketplaces**: Amazon, eBay, Walmart
- **Wholesale**: B2B customers, distributors
- **Retail**: Physical stores
- **Custom**: Any other sales avenue

## Why Track Channels?

| Benefit | Description |
|---------|-------------|
| **Demand Forecasting** | Forecast by channel for accuracy |
| **Inventory Allocation** | Know where demand comes from |
| **Performance Analysis** | Compare channel performance |
| **Pricing** | Channel-specific pricing |
| **User Access** | Control who sees what data |

## The Channels Dashboard

Navigate to **Channels** in the sidebar.

### Dashboard Features

| Feature | Description |
|---------|-------------|
| **Channel List** | All channels with status |
| **Search** | Find channels quickly |
| **Create Button** | Add new channels |
| **Status Indicators** | Active/Archived status |

### Channel Information

Each channel shows:

- **Name**: Channel display name
- **Type**: Category of channel
- **Status**: Active or Archived
- **SKU Count**: Products available on channel

## Channel Types

Organize channels by type:

| Type | Examples |
|------|----------|
| **E-commerce** | Shopify, WooCommerce, BigCommerce |
| **Marketplace** | Amazon, eBay, Etsy |
| **Wholesale** | B2B portal, distributor orders |
| **Retail** | Physical store POS |
| **Direct** | Phone orders, manual entry |

## Creating a Channel

<Steps>
  <Step title="Navigate to Channels">
    Go to **Channels** in the sidebar
  </Step>

  <Step title="Click Create Channel">
    Click the **Create Channel** button
  </Step>

  <Step title="Enter Details">
    | Field | Required | Description |
    |-------|----------|-------------|
    | **Name** | Yes | Channel display name |
    | **Code** | Yes | Short identifier |
    | **Type** | No | Channel category |
    | **Description** | No | Additional details |
  </Step>

  <Step title="Save">
    Click **Save** to create the channel
  </Step>
</Steps>

## Channel Status

Channels can be Active or Archived:

| Status | Behavior |
|--------|----------|
| **Active** | Used in forecasting, available for orders |
| **Archived** | Hidden from active use, data preserved |

## Channels and Forecasting

Channels are essential for demand forecasting:

### Forecast by Channel

- Each SKU/channel combination has its own forecast
- Different channels may have different demand patterns
- Promotions can be channel-specific

### Channel Performance

Track metrics per channel:

- Sales volume
- Forecast accuracy
- Growth trends

## Channels and Inventory

### Fulfillment Mapping

Configure which warehouses fulfill which channels:

| Channel | Fulfilled From |
|---------|----------------|
| Shopify US | Main Warehouse |
| Amazon | Amazon FBA |
| Wholesale | Regional DCs |

### Channel Availability

Control which SKUs are available on which channels:

- All SKUs on all channels
- Channel-specific assortments
- Exclusive products

## Channel Access Control

Control user access by channel:

| Role | Access |
|------|--------|
| **Admin** | All channels |
| **Sales Rep** | Assigned channels only |

See [User Access](/administration/user-access) for configuration.

## Best Practices

<AccordionGroup>
  <Accordion title="Consistent Naming">
    Use clear, consistent channel names:
    - Include region: "Shopify US", "Shopify EU"
    - Include type: "Amazon Marketplace", "Wholesale B2B"
  </Accordion>

  <Accordion title="Match Business Structure">
    Create channels that match how you manage the business:
    - Separate channels for separate P&Ls
    - Group similar channels if managed together
  </Accordion>

  <Accordion title="Consider Forecasting Needs">
    Create granularity needed for accurate forecasts:
    - Different patterns = different channels
    - Same patterns = can combine
  </Accordion>

  <Accordion title="Plan for Growth">
    Design for scalability:
    - Room for new channels
    - Consistent structure
  </Accordion>
</AccordionGroup>
